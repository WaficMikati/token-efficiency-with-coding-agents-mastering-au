# 01.0 How Token Usage Works in Coding Agents

Understanding how tokens are used during code generation is essential for working effectively with AI coding agents like **GitHub Copilot** and **Cursor**. Let's explore what happens behind the scenes and why managing tokens wisely can save you time and resources.

## What Are Tokens?

Tokens are the building blocks of text that AI models process. They can be whole words, parts of words, or even punctuation marks. When you ask a coding agent to generate code, it consumes tokens both in your input prompt and in the output it produces.

## How Tokens Are Consumed

- **Input Tokens:** These are the tokens in the code or text you provide to the agent as context or instructions.
- **Output Tokens:** These are the tokens generated by the agent as code suggestions or completions.

Every token processed contributes to the total token count, which affects the speed and cost of using the AI tool.

## Why Managing Tokens Matters

- **Cost Efficiency:** Many AI services charge based on the number of tokens used. Using fewer tokens means lower costs.
- **Performance:** Excessive tokens can slow down response times and make the coding experience less smooth.
- **Relevance:** Efficient token use helps the agent focus on the most important context, improving the quality of suggestions.

## Practical Example

Imagine you want the agent to write a function. Providing a concise, clear prompt with only necessary context uses fewer input tokens. The agent then generates the function code using output tokens. If your prompt is too long or includes irrelevant information, you waste tokens without improving results.

## Summary

By understanding token usage, you can write prompts and interact with coding agents more strategically, balancing the amount of information you provide with the quality of code generated. This skill is key to becoming a more efficient and cost-effective developer.
